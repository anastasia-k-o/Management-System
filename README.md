ZIL Cultural Center Management System

ZIL Cultural Center Management System — это десктопное приложение, разработанное на Python с использованием PyQt5 и SQLAlchemy для автоматизации всех ключевых процессов Культурного центра ЗИЛ. Система охватывает три основных направления деятельности: развлекательное, образовательное и культурно-просветительское, и обеспечивает полный цикл управления — от планирования мероприятий до учёта продаж и документооборота выставок.

Основные возможности
1. Управление пространствами
   - Создание и редактирование пространств (залы, студии, выставочные площади).

   - Хранение информации: наименование, вместимость, описание.

   - Возможность указать, что пространство состоит из локаций (например, партер, балкон). Для каждой локации задаётся количество рядов и мест в ряду.

   - Автоматическая фильтрация пространств при создании мероприятия: отображаются только те, вместимость которых >= планируемого количества посетителей.

2. Развлекательная деятельность (Мероприятия)
   - Типы мероприятий — настраиваемый справочник.

   - Регистрация мероприятия: дата, название, тип, время начала и окончания, количество посетителей, пространство, признак платности, описание.

   - Для платных мероприятий — интерактивная установка цен на билеты:

      - Визуальное отображение схемы зала (по локациям, рядам, местам).

      - Возможность выделить группу мест мышкой и задать для них единую стоимость.

   - Сохранение цен для каждого места.

   - Продажа билетов:

     - Выбор платного мероприятия.

     - Отображение занятых/свободных мест.

     - Продажа одного или нескольких билетов за одну операцию.

     - Автоматическая подстановка цены для выбранного места.

   - Отчёты по продажам:

      - За произвольный период (неделя, месяц, даты).

      - Количество проданных билетов и выручка по мероприятиям.

      - Детализация по ценовым категориям (сколько билетов продано по каждой цене).

3. Образовательная деятельность
    - Студии (наименование, описание).

    - Преподаватели (ФИО).

    - Посетители центра (ФИО) — база учеников.

    - Приказ о работе студии:

      - Выбор студии, преподавателя, периода работы (даты начала и окончания).

      - Дни недели и время проведения занятий.

      - Проверка занятости преподавателя: система не позволит создать приказ, если в указанные дни и время преподаватель уже ведёт другую студию.

    - Заявки на посещение студии:

      - Регистрация желающих учеников с привязкой к конкретному приказу.

      - Из приказа можно просмотреть все заявки по данной студии и периоду.

    - Установка цен на абонементы:

      - Привязка к приказу о работе студии.

      - Задание цены за разовое занятие вручную.

      - Автоматический расчёт стоимости абонемента на месяц и на год по заданному правилу (например, скидка за опт).

      - Просмотр цен непосредственно из карточки приказа.

    - Продажа абонементов:

      - Выбор заявки (ученика) и приказа.

      - Выбор вида абонемента (разовый, месяц, год) — цена подставляется автоматически.

      - Фиксация продажи с датой и суммой.

    - Отчёты по образовательной деятельности:

      - Набор в студии: название студии, перечень записанных участников за период.

      - Продажи абонементов: по студиям, периодам — количество проданных абонементов каждого вида, итоговая сумма.

4. Культурно-просветительская деятельность (Выставки)
  Экспонаты:
  
    - Наименование, владелец (выбор из: Культурный центр, студия, сторонняя организация).
  
    - Сторонние организации (наименование) — справочник.
  
    - Выставки:
  
    - Название, тип (внутренняя / внешняя), описание.
  
    - Приказ о проведении выставки:
    
      - Дата формирования, выбор выставки, даты проведения, место проведения (пространство).
    
    - Автоматическое формирование перечня экспонатов и их владельцев на основе типа выставки:
    
      - Для внутренней выставки доступны только экспонаты, принадлежащие Культурному центру или студиям.
    
      - Для внешней — только экспонаты сторонних организаций.
    
    - Поступление экспонатов от сторонней организации (только для внешних выставок):
    
      - Дата поступления, выбор приказа о проведении выставки.

    *Перечень экспонатов (копируется из приказа).

    *После оформления поступления экспонаты становятся доступны для бронирования.

     - Акт передачи экспонатов на выставку:

        - Дата передачи, приказ о проведении выставки.

        - Перечень экспонатов (из приказа). Передать можно только те экспонаты, которые:

           - для внутренней выставки: принадлежат центру/студиям и ещё не забронированы;
          
           - для внешней: поступили от сторонней организации и не забронированы.
          
          После подписания акта экспонаты блокируются для других выставок.

    - Акт возврата экспонатов:
    
        - Дата возврата, приказ.
    
        - Перечень возвращаемых экспонатов (все, переданные на выставку).
    
    - Контроль организации выставки:
    
       - Для каждого приказа отображается «карта маршрута» с этапами: Приказ → Поступление (если внешняя) → Передача → Возврат.
      
       - Текущая стадия выделяется, система не позволяет пропускать шаги (например, нельзя оформить передачу без приказа).

5. Загрузка данных из Excel
  - Поддержка импорта начальных данных:
  
    - Пространства
  
    -  Студии
  
    -  Экспонаты
  
    - Преподаватели
  
  *Шаблоны Excel предоставляются в комплекте.

Технологический стек
  - Язык: Python 3.9+
  
  - GUI: PyQt5
  
  - ORM: SQLAlchemy (Core + ORM)
  
  - База данных: SQLite (по умолчанию) / PostgreSQL (опционально)
  
  - Работа с Excel: openpyxl / pandas
  
  - Отчёты: ReportLab (PDF), matplotlib (для графиков)
